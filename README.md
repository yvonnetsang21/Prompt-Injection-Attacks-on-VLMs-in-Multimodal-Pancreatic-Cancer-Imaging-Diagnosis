# 胰腺影像场景下多模态大模型的安全性与鲁棒性（作图代码）

这里是与本项目/论文配套的 **GitHub 仓库**。和不少涉及**对抗设置、提示注入或安全评估**的工作类似，我们在审稿阶段先公开**与论文图表直接对应的 R 作图脚本**，方便核对统计图的数据处理与呈现方式；完整可复现实验代码会在**论文正式发表（或期刊要求的代码公开节点）**后一并整理上传，并在这里更新说明与链接。

## 这篇工作在做什么

在**胰腺相关医学影像**的多任务设定里，我们考察视觉–语言模型在不同配置下的表现，并讨论提示注入、模态与张数、序列与打乱，以及防御策略等对结果的影响。具体定义、协议和结论请以正式稿件为准；本仓库侧重「图是怎么画出来的」，而不是完整实验流水线。

## 仓库里有什么、暂时没有什么

**目前包含：** `Data_visualization_code/` 下的 R 脚本（Baseline、Attack、Defense、模态与张数、序列与打乱等相关图）。

**暂不包含：** 数据构建、图像注入生成、模型 API 调用、大规模实验编排与评判等**端到端实验代码**。审稿阶段我们采取的是领域里很常见的**折中**：先公开作图脚本，保证图表的数据与呈现逻辑可查；完整流水线等录用后再放出，减少「未正式发表就把可一键复现的攻击细节完全公开」的风险——和安全/对抗机器学习论文里常见的**分阶段代码公开**节奏类似。

若审稿人或编辑部需要额外材料，请通过稿件中的联系方式沟通，我们会在合规前提下配合。

## 环境

- R 建议 4.2 及以上  
- 依赖包：`reshape2`、`ggplot2`、`dplyr`、`ggpubr`、`ggthemes`

```r
install.packages(c("reshape2", "ggplot2", "dplyr", "ggpubr", "ggthemes"))
```

## 如何跑起来

脚本读取的是已经**聚合好的 CSV**（路径多相对于你运行 R 时的当前工作目录，例如各 `*_performance.csv`、`case_level_data.csv`，以及 `Defense.R` 里配置的子目录等）。把与正文/补充材料一致的汇总表放到对应位置，或按本地路径改一下脚本里的常量即可。若系统缺中文字体，导出图时可能需要按环境微调 `ggsave` 或主题里的字体设置。
